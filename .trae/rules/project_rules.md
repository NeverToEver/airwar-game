# 编程架构与工程化规范（全局强制）

**所有代码任务前自动激活。**

## 核心原则

**单一职责**：类/函数只做一件事  
**高内聚低耦合**：相关逻辑集中，模块间通过接口通信  
**面向接口**：外部依赖抽象，不依赖具体实现  
**可维护/可扩展**：结构清晰，避免硬编码、深层嵌套

## 项目结构

- 按功能模块划分，入口文件不含业务逻辑
- 核心模块独立：窗口、渲染、输入、游戏逻辑、资源管理
- 配置统一管理

## 代码规范

- 命名清晰（除i/j循环计数器外）
- 函数≤40行，超出必须拆分
- 最多3层嵌套，用早返回/卫语句
- 重复代码≥3次必须抽离
- 全局变量最小化，状态集中管理

## 解耦要求（游戏项目尤其严格）

- 窗口/渲染/事件封装独立模块
- 外部不直接操作底层库（如pygame）
- 所有底层操作通过类接口暴露
- 游戏逻辑、UI、输入、渲染相互隔离

## 自动执行

1. 写代码前先做架构设计
2. 识别坏味道：散落逻辑、重复代码、混合职责
3. 提出重构建议并给可运行代码
4. 生成模块设计文档、接口说明、维护指南

## 输出要求

- 先说明设计思路与模块划分
- 再给完整可运行代码
- 确保代码可直接运行、结构干净、便于扩展
