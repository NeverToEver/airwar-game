#### 1.2.2 子弹更新时机方案

**原问题**: `bullet.update()`在碰撞检测前调用

**修复方案逻辑**:
1. 在`GameScene._update_game()`中先更新所有子弹
2. 在`CollisionController`中不再更新子弹

**逻辑验证**:
```python
# GameScene._update_game()更新顺序
def _update_game(self):
    self.player.update()          # 1. 玩家更新（不含子弹更新）
    self.player.auto_fire()       # 2. 发射子弹
    self._update_all_bullets()    # 3. [新增] 更新所有子弹
    self._update_entities()       # 4. 敌人更新
    self._check_collisions()      # 5. 碰撞检测（使用已更新的子弹）
```

**结论**: 更新顺序正确，确保碰撞检测使用最新位置。

#### 1.2.3 GameOver触发方案

**原问题**: `player.active`立即变为False

**修复方案逻辑**:
1. 添加`GameplayState`枚举（PLAYING/DYING/GAME_OVER）
2. 玩家死亡时设置状态为DYING，启动死亡计时器
3. 计时器归零时设置状态为GAME_OVER

**逻辑验证**:
```python
# 状态转换流程
健康 > 0 → 受伤 → 无敌状态（PLAYING）
健康 = 0 → 受伤 → 死亡动画（DYING，90帧）
DYING计时器 = 0 → GAME_OVER
```

**结论**: 状态机逻辑清晰，避免立即触发GameOver。
