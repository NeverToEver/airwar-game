## 第三部分：项目架构分析

### 3.1 整体架构

AIRWAR项目采用**分层架构**设计，核心架构图如下：

```
airwar/
├── config/              # 配置层
├── data/                # 数据层
├── entities/            # 实体层
├── game/                # 游戏核心
│   ├── buffs/          # 天赋系统
│   ├── controllers/    # 控制器
│   ├── mother_ship/    # 母舰系统
│   ├── rendering/      # 渲染系统
│   └── systems/        # 游戏系统
├── input/               # 输入层
├── scenes/              # 场景层
├── ui/                  # UI层
├── utils/               # 工具层
├── window/              # 窗口层
└── tests/               # 测试层
```

### 3.2 架构层次详解

#### 3.2.1 配置层 (config/)

|------|------|----------|

### 3.3 架构原则遵循情况

| 原则 | 描述 | 实施状态 | 评分 |
|------|------|----------|------|

| LSP (里氏替换) | 子类可替换父类 | ✅ 良好遵循 | 92/100 |

### 7.3 遗留问题详情

#### Issue #004: cycle_count在逃跑时未更新

**问题描述**:
当Boss被击杀时，`game_controller.cycle_count += 1`会增加游戏进度。当Boss逃跑时，这个计数器不会增加，导致逃跑的Boss不会推进游戏进度。

**建议方案**:
- **方案A**: 逃跑时也增加cycle_count（平衡性可能受影响）
- **方案B**: 保持现状（当前实现）- 逃跑不增加进度是合理的惩罚机制
