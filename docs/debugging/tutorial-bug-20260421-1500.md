# Debugging Session: Tutorial Scene Issues

## Date: 2026-04-21
## Issue: Tutorial Scene - Events Not Working, Layout Broken

---

## Problem Statement

**System Context:**
- Game: Air War (Pygame-based)
- Issue Location: `TutorialScene` in `airwar/scenes/tutorial_scene.py`
- User Reports:
  1. ESC key doesn't exit tutorial
  2. Back button doesn't work
  3. Cannot close window
  4. Layout is cramped and misaligned

**Expected Behavior:**
- ESC should immediately exit tutorial and return to menu
- Enter/Space should also exit
- Mouse click on button should exit
- Layout should display all controls clearly without overlap

---

## Root Cause Analysis

### Root Cause 1: Layout Overflow
**Finding:** Panel size was insufficient for content
- Original: 700x650 pixels
- Content required: ~750+ pixels total height
- Result: Content overflowed beyond panel boundaries

**Fix:** Redesigned layout with proper sizing
- New panel: 500x580 pixels
- Consistent with MenuScene styling
- Option rows: 60px height, 8px gap

### Root Cause 2: Event Handling Indirect Call
**Finding:** Events were being routed through SceneManager
- `_handle_scene_events()` → `scene_manager.handle_events()` → `current_scene.handle_events()`
- Potential for event filtering or blocking in the chain

**Fix:** Direct event handling in tutorial flow
- Bypass SceneManager for tutorial scene
- Direct call to `tutorial_scene.handle_events(event)`
- Added debug logging to trace event flow

### Root Cause 3: Complex Content Structure
**Finding:** Original design with sections, key boxes, and detailed descriptions
- Too many visual elements
- Poor alignment between keys and descriptions

**Fix:** Simplified content structure
- Flat list of control items
- Key on left, description on right
- Clean MenuScene-style rows

---

## Solutions Implemented

### 1. Layout Redesign
```python
# Before (problematic)
base_panel_width = 700
base_panel_height = 650
base_section_gap = 140

# After (fixed)
base_panel_width = 500
base_panel_height = 580
base_option_height = 60
base_option_gap = 8
```

### 2. Event Handling Fix
```python
# Before (indirect)
self._handle_scene_events(events)  # Via SceneManager

# After (direct)
for event in events:
    tutorial_scene.handle_events(event)  # Direct call
```

### 3. Debug Logging Added
```python
# In scene_director.py
for event in events:
    if event.type == pygame.KEYDOWN:
        print(f"[TUTORIAL DEBUG] KEYDOWN: key={event.key}")

# In tutorial_scene.py
if event.key == pygame.K_ESCAPE:
    print("[TUTORIAL SCENE] ESC pressed, setting running=False")
```
