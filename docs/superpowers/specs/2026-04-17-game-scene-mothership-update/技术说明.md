# GameScene 与母舰系统更新技术说明

> **文档版本**: 1.0  
> **编制日期**: 2026-04-17  
> **更新类型**: 架构整理、状态机修复、存档恢复增强  
> **影响范围**: `GameScene`、母舰系统、存档模型、包初始化  

---

## 一、更新背景

本次更新聚焦两个问题域：

1. `GameScene` 运行阶段过于集中，输入、碰撞、子弹清理、奖励弹窗、母舰状态切换混杂在同一条更新链路中，导致行为边界不清晰。
2. 母舰系统在停靠、离舰、存档恢复、窗口缩放等场景下存在状态不同步问题，尤其是 `UNDOCKING` 阶段的按键释放、停靠恢复、旧存档兼容与运行时数据丢失。

本次改动遵循以下基本原则：

- **单一职责**: 将 `GameScene` 的“输入门控”“战斗更新”“停滞状态更新”“存档恢复”拆成明确方法。
- **最小暴露面**: 包初始化改为惰性导出，避免仅导入逻辑模块时被 `pygame` UI 依赖牵连。
- **状态一致性**: 母舰状态、玩家控制权、UI 可见性、存档字段保持统一语义。
- **向后兼容**: 存档读取增加字段兜底与类型清洗，兼容旧存档与异常数据。

---

## 二、GameScene 优化说明

### 2.1 更新循环重组

`GameScene.update()` 现在按以下顺序执行：

1. 奖励面板动画更新
2. 母舰集成器更新
3. 入场动画更新
4. 母舰停靠/动画中的停滞态处理
5. 暂停态与奖励选择态拦截
6. 正常战斗态更新

这样处理后，不同运行阶段的边界更加清晰，避免了旧实现中“部分系统更新、部分系统跳过”的混乱。

### 2.2 输入门控

新增 `_can_accept_combat_input()`，统一约束玩家在以下场景中的射击输入：

- 入场动画期间
- 暂停期间
- 奖励选择期间
- 母舰已停靠时
- 母舰动画导致玩家控制被禁用时

修复结果：

- 停靠/离舰过程中不会再错误发射子弹
- 入场动画阶段不会再接收战斗输入

### 2.3 子弹与碰撞修复

本次同步修复了多个运行时问题：

- 玩家子弹与敌方子弹清理改为安全过滤，避免遍历时删除导致漏清理
- 敌机碰撞玩家时，改为实际扣血，而非仅设置无敌状态
- Boss 击杀分数不再重复累计
- 爆炸子弹会先对直接命中目标结算伤害，再处理范围伤害
- `kills` 属性修正为映射真实击杀数，而非 `cycle_count`

### 2.4 存档恢复增强

`GameScene.restore_from_save()` 现在会恢复：

- 分数、击杀数、Boss 击杀数
- `cycle_count` 与 `milestone_index`
- 玩家生命、攻击力、射速、射击模式、速度
- 已解锁 buff 与 buff 等级

并按场景分支恢复：

- 若存档处于母舰中，则恢复到稳定的停靠态
- 若存档处于正常战斗中，则恢复到有效飞行位置，并跳过入场动画

## 三、母舰系统修复说明

### 3.1 状态机修复

`MotherShipStateMachine` 的 `H_RELEASED` 事件现在只在 `PRESSING` 状态下生效。

修复前问题：

- 玩家从 `DOCKED` 按下 `H` 进入 `UNDOCKING`
- 松开按键时又被错误处理成回到 `IDLE`
- 随后 `UNDOCKING_ANIMATION_COMPLETE` 无法触发正常恢复

修复后：

- `UNDOCKING` 阶段不会被按键释放中断
- 离舰动画完成后可稳定回到 `IDLE`

### 3.2 集成器职责收敛

`GameIntegrator` 增加并整理了以下职责：

- `_set_player_position()` 统一动画与恢复时的玩家位置写入
- `_reset_input_state()` 统一重置输入检测器状态
- `restore_docked_state()` 统一恢复停靠态
- `resize()` 统一同步母舰与进度条 UI 的尺寸信息

同时修复了原有缺陷：

- 清空敌人时不再向只读 `boss` 属性赋值，而是直接操作 `spawn_controller.boss`
- 停靠与离舰阶段的 UI 可见性和控制权语义更加一致
- `SAVE_GAME_REQUEST` 不再额外篡改 `GameScene.paused`

### 3.3 输入检测器修复

`InputDetector` 新增 `reset()`，用于在停靠、离舰、恢复存档时清空内部按键状态和读条状态，避免旧输入残留影响新状态。

### 3.4 存档模型增强

`GameSaveData` 新增并校验以下字段：

- `milestone_index`
- `boss_kill_count`
- `player_x`
- `player_y`
- `player_bullet_damage`
- `player_fire_interval`
- `player_shot_mode`
- `player_speed`

新增安全清洗逻辑：

- 数值字段统一做类型转换与最小值限制
- `difficulty` 与 `shot_mode` 做合法值校验
- `is_in_mothership` 做安全布尔解析
- 异常字段自动回退到可用默认值

## 四、配套优化

### 4.1 玩家武器冷却模型

玩家实体新增 `shot_cooldown_frames`，用于表示武器的实际射击间隔。

修复前：

- `Rapid Fire` 只会影响当前倒计时值
- 后续每次射击仍固定重置为 `8` 帧

修复后：

- 射击行为统一使用可配置冷却间隔
- `Rapid Fire` 的提升可以稳定体现在持续输出上

### 4.2 包初始化降耦

以下包初始化已改为惰性导出：

- `airwar.game`
- `airwar.game.mother_ship`

目的：

- 降低逻辑模块对 `pygame` UI 模块的隐式耦合
- 让纯逻辑测试可以在更轻的导入链路下运行
